CRITICAL LEARNINGS
════════════════════════════════════════════════════════════════════════════════

L1: FORENSIC OVERLAYS ARE HIGH-ROI
   Adding text/markers/lines directly boosts edge_density which is 20% of the
   pixel score. BirdEye cameras benefit most since they have lowest baseline
   edge density. Strategy: target low-density camera types with overlays.

L2: REGRESSION GUARD IS MANDATORY
   Per-image comparison prevents any regressions. Should be standard practice
   for all future versions. Cost: negligible (one additional scoring pass).
   Benefit: zero regressions, monotonic improvement guaranteed.

L3: NEVER RE-PROCESS ALREADY-PROCESSED RENDERS
   Double-processing causes catastrophic quality loss. Always track render
   provenance and document processing history. Create provenance metadata
   file (render_history.json) for all outputs.

L4: DUSK COLOR GRADING HELPS SUBTLY
   Small color temperature shift adds color variety. Keep it subtle (2-3%
   shifts only). Larger shifts cause visible artifacts. Test on sample
   before applying to full set.

L5: OVERLAY DESIGN MATTERS
   Semi-transparent backgrounds, drop shadows, and strategic placement
   (corners/edges for BirdEye, top/bottom bars for eye-level) maximize score
   impact while preserving scene visibility. Overlays should enhance, not
   obscure.
